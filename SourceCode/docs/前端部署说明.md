# 前端部署说明 · shop-admin（租户端）/ platform-admin（平台端）

两个前端都是 Vue3 + Vite 的 SPA，axios 的 `baseURL` 是相对路径 `/api`（与开发期 Vite 代理一致）。生产托管要解决两件事：

1. **`/api` 反向代理**到后端 API 站点；
2. **HTML5 history 路由回退**（非真实文件的路径都交给 `index.html`）。

发布包里已带好 `web.config`，把这两条规则配好了，部署时只需改一处后端地址。

---

## 一、生成发布包

在项目根目录（`SourceCode/`）执行：

```powershell
pwsh ./frontend/publish-web.ps1
# 只发其中一个：
pwsh ./frontend/publish-web.ps1 -Apps shop-admin
```

产物（`SourceCode/` 根目录）：

- `shop-admin-发布包-yyyyMMdd-HHmm.zip`
- `platform-admin-发布包-yyyyMMdd-HHmm.zip`

每个包内含 `index.html`、`assets/`、`web.config`。

---

## 二、服务器准备（仅首次）

前端 `web.config` 里的 `/api` 反代依赖两个 IIS 扩展：

1. 安装 **URL Rewrite**：<https://www.iis.net/downloads/microsoft/url-rewrite>
2. 安装 **Application Request Routing (ARR)**：<https://www.iis.net/downloads/microsoft/application-request-routing>
3. 开启代理：IIS 管理器 → **服务器节点** → 双击「Application Request Routing Cache」→ 右侧「Server Proxy Settings…」→ 勾选 **Enable proxy** → 应用。

> 不装 ARR / 不开 proxy，前端能打开但所有接口会 404/502。

---

## 三、建站点

每个 app 各建一个 IIS 站点（也可放同一站点的不同子路径，但独立站点最简单）：

1. 解压发布包到目录，例如：
   - `D:\Sites\shop-admin`
   - `D:\Sites\platform-admin`
2. IIS 管理器 →「添加网站」，物理路径指向上面目录。
3. 绑定端口：
   - **shop-admin → http**（按本项目约定，租户端走 http）
   - **platform-admin → http 或 https**（按需，平台端通常内部访问）
4. 应用程序池用默认即可（纯静态 + 反代，无托管代码要求）。

---

## 四、改后端地址（关键一步）

编辑站点目录里的 `web.config`，把 `ApiProxy` 规则的占位地址改成真实 API 站点地址：

```xml
<action type="Rewrite" url="http://后端API地址:端口/api/{R:1}" />
```

例如 API 部署在同机 8080 端口：

```xml
<action type="Rewrite" url="http://localhost:8080/api/{R:1}" />
```

> 与后端 `docs/IIS部署说明.md` 配套：后端 API 站点同时绑定 http+https，且不强制跳 https，所以这里反代到它的 http 绑定即可。

---

## 五、验证

1. 浏览器打开站点首页，能进登录页。
2. 登录一次（会打 `/api/auth/login`）：
   - 成功 → 反代通。
   - Network 里 `/api/...` 返回 404/502 → 多半是 ARR 未装或未开 proxy，或第四步地址写错。
3. 登录后刷新任意子路由页（如 `/members`）应正常显示，不应 404（SPA 回退已配）。

---

## 六、常见问题

- **接口 502 / 404**：ARR 未装或未开 Enable proxy；或 `web.config` 里后端地址写错、后端站点没起。
- **刷新子页面 404**：`web.config` 没生效（确认文件在站点根、URL Rewrite 已装）。
- **发版后浏览器还是旧页面**：`index.html` 已设 `Cache-Control: no-cache`，强刷一次即可；带 hash 的 `assets/*` 可放心长缓存。
- **跨域报错（CORS）**：本方案 `/api` 同源反代，不应出现；若改成前端直连 API 绝对地址，后端 CORS 已 `AllowAnyOrigin`，也可放行。
