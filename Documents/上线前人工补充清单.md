# 上线前人工补充清单

> 本清单列出**代码层面无法替代、需要人工完成**的事项：账号凭证、基础设施、硬件、环境与决策。
> 系统功能开发已完成（后端 API / CS 收银端 / BS 管理端 / 平台端 / 小程序），以下为投产准备工作。
>
> 最后更新：2026-05-19

## 一、第三方账号与凭证

- [ ] **微信小程序**：注册小程序账号，取得 `AppId` / `AppSecret`，填入 API 的 `appsettings`（`Notifications:WeChat`）
- [ ] **订阅消息模板**：在微信公众平台申请 4 个模板（生日提醒 / 会员卡到期 / 预约提醒 / 充值到账），把模板 ID 填进 `Notifications:WeChat:Templates`
- [ ] **小程序 `project.config.json`**：把 `appid` 改成正式 appid（当前是占位 / 测试号）
- [ ] **小程序服务器域名**：在微信后台「开发设置」把后端 API 域名加入 request 合法域名（必须 HTTPS）
- [ ] **微信支付 V3**：商户号、API 证书、APIv3 密钥
- [ ] **支付宝**：应用 AppId、应用私钥 / 支付宝公钥
- [ ] **支付回调地址**：配置公网可达的回调 URL，并确认在 `TenantStatusMiddleware` 白名单内
- [ ] **短信服务商**（如启用短信通知）：账号、签名、模板

## 二、基础设施

- [ ] **MySQL 8.0** 部署，建库 `massage_saas`，生产连接串写入 `appsettings.Development.json` 或环境变量（勿入库）
- [ ] **Redis 7** 部署（P1 阶段起必需）
- [ ] **服务器 / 域名 / 反向代理**、HTTPS 证书

## 三、安全（务必处理）

- [ ] **JWT 签名密钥**：生产环境换成强随机密钥，不能沿用仓库默认值
- [ ] **默认平台管理员**：首次启动会 seed `admin / admin@123`，登录后**立即改密码**
- [ ] **AutoMapper 12.0.1 漏洞（GHSA-rvv3-g6hj-g44x）**：决策是否升级到 14+（有破坏性变更，见 `CLAUDE.md`）

## 四、硬件对接

- [ ] **小票打印机**：接好后改 `MassageSaas.Cs/appsettings.json` 的 `Printer` 节（`enabled:true`、串口号 + 波特率，或网口 IP + 9100），用收银台「测试打印小票」按钮现场核对出纸 / 切纸 / 钱箱
- [ ] **客显、来电盒、磁条读卡器**：目前是占位实现（不驱动硬件）。需硬件到位 + 厂商通讯协议，才能写真实驱动实现类替换占位（接口已就绪，只换 DI 注册）

## 五、部署与运维

- [ ] **健康检查**：把 `/health` 接入监控系统（Uptime Kuma / Prometheus 等）
- [ ] **日志**：Serilog 写 `logs/` 目录，配置归集与轮转保留策略
- [ ] **数据库备份**：制定备份与恢复方案
- [ ] **集成测试**：`MassageSaas.IntegrationTests` 当前用 InMemory 可直接跑；如需用 `Testcontainers.MySql` 跑真实 MySQL 端到端，CI / 本地需有 **Docker** 环境

## 六、业务初始化数据

- [ ] **套餐定价**：`DbInitializer` 已 seed 基础 / 标准 / 旗舰三档价格，需业务确认是否调整
- [ ] **各租户开店后录入**：门店、服务项目、房间、技师档案（含盲人资质证号、钟数上限）、提成规则、薪资档案

## 七、验收测试（需真机 / 真人）

- [ ] **小程序无障碍**：真机 + 读屏软件实测盲人技师 / 顾客流程（字号、aria-label、金额朗读）
- [ ] **CS 端无障碍**：用 NVDA / Windows 讲述人实测全键盘可达与控件朗读
- [ ] **微信通知闭环**：配置完模板后，真机走一遍「绑卡 → 充值 → 收到订阅消息」
- [ ] **支付闭环**：真实小额支付 + 回调验证幂等
