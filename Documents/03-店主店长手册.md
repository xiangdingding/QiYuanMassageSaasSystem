# 店主 / 店长手册（BS 端 Web）

> 适用对象：按摩店店主（ShopOwner）与店长（StoreManager）。
> 系统入口：按摩店 Web —— `https://shop.your-domain.com`（生产）/ `http://localhost:5174`（开发）。
> 默认账号：由平台管理员在开通租户时下发的临时账号。**首次登录立即改密**。
>
> 配套文档：《04-收银员手册.md》《05-技师小程序手册.md》《07-常见问题与故障处理.md》。

## 1. 角色边界

| 看得到 | 看不到 |
|---|---|
| **店主**：本租户下所有门店、所有数据；唯一可以「门店管理」「订阅状态」 | 别家租户的数据 |
| **店长**：当前登录绑定的那一家门店；不能开新门店、不能管订阅 | 别家门店 + 别家租户 |

> 想看其他门店的数据，店主用顶栏「门店切换」下拉。店长账号绑定到具体门店，由店主创建时指定。

## 2. 主菜单（20 项）

侧栏自上而下：

| 菜单 | 角色可见 | 主要用途 |
|---|---|---|
| 收银台 | ShopOwner / StoreManager / Cashier | 开单结账 —— 详见《04-收银员手册.md》 |
| 预约管理 | 同上 | 顾客线上/电话预约的处理 |
| 订单流水 | 同上 | 查历史订单、反结账、加钟 |
| 房间管理 | 同上 | 房间状态（空闲/占用）、计时房 |
| 会员管理 | 同上 | 开卡/充值/查询/退卡/转赠 |
| 技师排队 | 全员（含技师本人） | 谁在排队、谁在服务、上下钟 |
| 日报与业绩 | POS 组 | 日/周/月/年 营业额、提成、客流、会员分析、服务热度、技师质量 |
| 日结/交班 | POS 组 | 当班现金对账、当班营业额、交班单 |
| 服务项目 | ShopOwner / StoreManager | 维护服务、单价、时长、提成基数 |
| 会员套餐 | POS 组 | 充 X 送 Y、计次卡、月季年卡 |
| 优惠券 | POS 组 | 团购券（美团/点评）核销规则 |
| 物耗库存 | POS 组 | 精油/毛巾/足浴包等耗材进出库与预警 |
| 服务评价 | POS 组 | 顾客评价列表、回复、好评率 |
| 投诉处理 | POS 组 | 改派记录、补偿、闭环 |
| 排班与请假 | ShopOwner / StoreManager | 技师排班、请假审批、跨店调动 |
| 提成规则 | 同上 | 阶梯提成、技师等级溢价 |
| 工资结算 | 同上 | 月度封盘、底薪+提成+奖金+加班+满勤 |
| 员工管理 | 同上 | 员工入职/离职/角色/资质证 |
| 门店管理 | 仅 ShopOwner | 新开/修改门店 |
| 订阅状态 | 仅 ShopOwner | 查到期日、续费、付款记录 |

> 收银员/技师角色看到的菜单是子集，详见各自手册。

## 3. 初次开店：业务初始化清单

平台管理员开通租户后，你拿到账号 → 登录 → 第一周要把以下数据录完，否则收银员开不了单。

### 3.1 改密 + 个人信息

右上头像 → 个人设置 → 改密、改姓名、改手机。

### 3.2 门店

侧栏 **门店管理 → 新建门店**：

- 门店名（如「明心朝阳店」）
- 地址、电话、营业时间
- 是否总店（每个租户只能有一个总店）

> 套餐里写的"门店数 = 1"指本租户最多 1 家门店。如果你买的基础版还想开第二家，去找平台管理员升级套餐。

### 3.3 服务项目

**服务项目 → 新建**：

| 字段 | 含义 |
|---|---|
| 编码 | 用于收银快速搜索（如 `Z60` = 中式 60 分钟） |
| 名称 | 显示名（中式按摩 60 分钟） |
| 时长 | 60 分钟，影响排队预估 |
| 单价 | 散客价 |
| 会员价 | 持卡会员单价（低于单价） |
| 提成基数 | 用于工资计算，可与单价不同 |
| 类别 | 中医/足疗/SPA…，用于报表分类 |
| 是否启用 | 季节性产品可临时下架 |

### 3.4 房间

**房间管理 → 新建房间**：

- 房间号（如 `A1`）
- 类型：标准间 / VIP / 包房 / 计时房
- 计时房需设小时单价（按时间计费不按服务计费）
- 关联门店（店主跨店开店时区分）

### 3.5 员工

**员工管理 → 新增**：

| 字段 | 含义 |
|---|---|
| 用户名 + 临时密码 | 登录用 |
| 真实姓名 + 手机 | 必填 |
| 角色 | 店长 / 收银员 / 技师 |
| 绑定门店 | 店长/收银员/技师必须绑定到一家门店 |
| 技师等级 | 初/中/高 —— 影响提成倍率 |
| 盲人资质证号 | 仅技师，盲人专属字段 |
| 钟数上限 | 仅技师，每日最大上钟数（过载告警） |
| 入职日期 | 用于工龄计算 |

### 3.6 提成规则

**提成规则 → 新建**：

支持阶梯：

- 按"月累计钟数"分档：0-100 钟 ×30%、100-200 钟 ×35%、200+ ×40%
- 按"技师等级"乘系数：初级 ×1.0、中级 ×1.1、高级 ×1.2
- 按服务类别独立规则：中医/足疗/SPA 分开

规则一旦生效会用于工资结算，**修改规则不追溯历史**——已封盘的工资单不重算。

### 3.7 物耗库存

**物耗库存 → 新增物料**：

- 编码 / 名称 / 单位（瓶/条/包）
- 安全库存（低于此值在「库存预警」里红色高亮）
- 单服务消耗量（可选）：如「中式按摩 1 次 = 1 包精油」，结账时自动扣库存

### 3.8 会员套餐（可选）

**会员套餐 → 新建**：

- 充值送：「充 1000 送 200」
- 计次卡：「10 次中式按摩，4800 元」
- 期限卡：「1 个月内不限次中式按摩，2800 元」
- 折扣卡：「会员全部 8 折」

## 4. 收银台

详见《04-收银员手册.md》。要点：

- 店主/店长**默认也能用**收银台（多角色复合权限）。技师角色看不到收银台菜单。
- 收银台 Web 端与 CS 端**功能完全一致**——白班可能 CS 端开单（带打印机/钱箱），夜班店长直接在 Web 端开单也行。
- 反结账（订单流水 → 选订单 → 反结账）会回滚库存、会员卡余额、提成；要谨慎，建议加密码或店长权限。

## 5. 预约管理

**预约管理**：

- 列表显示所有预约：到店时间、姓名、手机、人数、指定技师、备注、状态
- 状态：待到店 / 已到店转单 / 已取消 / 爽约
- 顾客在小程序提交的预约会直接进这个列表
- 「到店」按钮：把预约转换为订单 → 跳到收银台预填好
- 取消时填原因；爽约会在会员档案里累计（次数过多自动降级）

## 6. 房间管理

- **空闲 / 占用 / 清洁中** 三态
- 收银台开单时分配房间，结账后房间自动转「清洁中」，保洁手动点「已清洁」回到空闲
- **计时房**：开始 → 进入计时；客人按服务铃 / 收银台手动点「结束」→ 系统按小时单价计费
- 计时房的营业额并入日报「计时房收入」单列

## 7. 会员管理

### 7.1 开卡

**会员管理 → 新建**：

- 卡号（手动输入 / 自动生成）
- 持卡人姓名 + 手机（手机用于会员卡查询、微信通知绑定）
- 等级：白/银/金/钻 —— 由折扣等级决定（在「提成规则」附近的「会员等级」页配置）
- 初始余额（充值 + 赠送）
- 关联推荐人（用于返佣）

### 7.2 充值

- 在会员详情页 → 「充值」按钮
- 选金额、选支付方式（现金/微信/支付宝/银行卡）
- 自动叠加套餐里的「充 X 送 Y」赠送金额
- 当晚轮询服务会推送一条「充值到账」订阅消息给顾客微信（前提是顾客已绑卡）

### 7.3 转赠 / 退卡

- **转赠**：A 会员 → 把卡内剩余余额转给 B 会员
- **退卡**：扣除已使用部分（按充值时实际付现金计算），剩余退给会员（建议走对公账户转账）
- 两种操作都有审计：操作人、时间、金额、备注。审计在「会员详情 → 操作历史」可查。

### 7.4 卡到期提醒

- 系统每天自动扫描接下来 30 天内到期的会员套餐
- 对绑定微信的会员推送「会员卡到期提醒」订阅消息
- 同时在「会员管理」列表标红

## 8. 技师排队

**技师排队**：

- 在岗（OnDuty）/ 休息 / 下班 / 请假 / 培训 几种状态
- 顺序按"上一次下钟时间"升序——最久没上的优先
- 收银员开单时点「轮号」→ 取队首；点「指定」→ 选具体技师
- **过载提醒**：技师当日钟数 ≥ `钟数上限` 时，开单页会黄色告警；店长可以选择是否强制

## 9. 日报与业绩（9 个 Tab）

| Tab | 内容 |
|---|---|
| 日报 | 当日营业额、客流、平均单价、各通道收款 |
| 周报 | 周聚合、环比 |
| 月报 | 月聚合、同比 |
| 年报 | 年趋势 |
| 提成 | 按技师查当月累计、按月封盘 |
| 技师排行 | 钟数 / 营业额 / 评分三榜 |
| 会员分析 | 新增/活跃/沉睡、客单价、复购率 |
| 服务热度 | 哪个服务卖得多、哪个该停 |
| 客流 | 时段分布（小时） + 工作日 vs 周末 |

> 数据口径：所有报表都剔除「反结账」订单与小费（小费不入营业额）。
> 反结账后已下发提成的处理：提成会自动反向冲销，已封盘的工资不变（避免重算）。

## 10. 日结 / 交班

**日结/交班**：

- 当班开始 → 系统记入班时间
- 整班期间所有现金收/找零进入「现金应收」
- 当班结束 → 收银员点「交班」→ 输入实际现金数 → 系统比对差额
- 差额 > 0：长款；差额 < 0：短款。短款会在工资里扣（或店长免责）
- 生成「交班单」打印，店长签字

## 11. 服务评价

- 顾客在小程序「我的预约」里给当次服务打星 + 评价
- 系统按"评价时间"列出所有评价
- 你可以「回复」（公开，技师能看）
- 差评（≤2 星）自动进「投诉处理」队列

## 12. 投诉处理

- 来源：差评、电话投诉、收银员现场记录
- 流程：
  1. 接报 → 录入投诉单（投诉对象 = 哪位技师/服务）
  2. 调查 → 处理（改派 / 退款 / 赠送 / 警告 / 培训）
  3. 与顾客沟通 → 记录沟通结果
  4. 闭环
- 「改派」会触发当次订单的技师变更，新技师提成生效、原技师不计提成
- 投诉次数累计写入技师档案，影响晋升 / 试用期评估

## 13. 排班与请假

### 13.1 排班

**排班与请假**：

- 周视图：横轴日期、纵轴技师
- 拖拽指派班次（早/中/晚/全天）
- 跨店调动：选择目标门店 + 日期 → 该日期技师归属临时改到目标店，回来后自动恢复

### 13.2 请假

- 技师在小程序提交请假（如果开了该功能）；或店长代提
- 类型：事假 / 病假 / 产假 / 调休
- 审批：店长审批 → 通过则该天不计入「应出勤」、不扣满勤奖
- 请假记录用于工资结算的「考勤」项

## 14. 提成规则

见 3.6。日常变动较少；变动时务必通知财务，避免封盘错。

## 15. 工资结算

**工资结算**：

1. 每月 1 号系统自动生成"上个月工资单"草稿，状态 `Draft`
2. 你逐人审核，可以加备注、加奖金、扣调整
3. 全部审完后「批量封盘」→ 状态 `Locked` → 该月工资数据不可再改
4. 封盘后技师可以在小程序看到自己的工资明细
5. 「导出」可生成 Excel 给财务，或一键标记「已发放」

构成（每个项目都可在「提成规则」配置）：

```
到手工资 = 底薪 + 提成 + 奖金 + 加班 ± 满勤奖 ± 调整 - 扣款
```

**小费不入工资单**——单独在小程序「我的薪资 / 不入工资栏」展示。

## 16. 员工管理

见 3.5。日常变动：

- 入职：建账号 + 录档案
- 离职：「停用」（不删除，保留历史数据用于报表）
- 角色变更：店主有权改成店长/收银员等
- 资质证更新：盲人技师证书到期更新

## 17. 门店管理（仅店主）

跨店运营才会用。要点：

- 每新增一家门店要先在「员工管理」给它分配店长
- 关闭门店：先把所有员工调走 → 设「停业」
- 总店与分店：报表里"全店汇总"自动合并

## 18. 订阅状态（仅店主）

**订阅状态**：

- 显示当前套餐、到期日、剩余天数
- 「续费」→ 选时长 → 选支付方式：
  - 微信支付：扫码 → 到账后系统自动续期
  - 支付宝：同上
  - 线下汇款：联系平台管理员，提供汇款单号后由平台「线下激活」
- 「付款记录」列出所有历史续费订单

> 到期前 30/15/7/1 天系统会发邮件提醒店主。

## 19. 多门店切换

顶栏右上有「门店」下拉。店主默认看到所有门店的聚合；选某家店后后续菜单只显示该店数据。店长账号无此下拉（绑定到唯一一家店）。

## 20. 与小程序的关系

- 顾客小程序「预约」→ 进你的「预约管理」
- 顾客小程序「绑定会员卡 + 订阅消息授权」→ 你充值时他能收到微信通知
- 技师小程序「上下钟 / 看业绩 / 看工资」→ 数据完全等于 BS 端的「技师排队 / 工资结算」

## 21. 数据导出

报表页 + 工资页 + 会员列表 都支持「导出 Excel/CSV」。导出文件包含本租户数据，**不可外泄给非本店人员**（含平台方）。

## 22. 备份与误操作恢复

- 数据库每天 0:00 全量备份（运维负责）。误删 / 误操作可联系运维恢复
- **请尽量用「停用」而不是「删除」**——停用保留历史数据用于报表
- 反结账后又想恢复：联系运维查 `audit_logs` + 备份
