## 1. 总览：上线时间线

```
T-30 天  ┬─ 第三方账号申请（微信小程序 / 微信支付 / 支付宝 / 短信）   ← 启动
         │  ※ 审核周期 1-7 工作日，必须最早启动
         │
T-14 天  ┼─ 基础设施部署（服务器/MySQL/Redis/域名/HTTPS）
         │  ├ 后端 API 部署到测试环境
         │  └ 前端三端构建产物部署
         │
T-7 天   ┼─ 安全加固（JWT 密钥、默认密码、漏洞复核）
         │  ├ 业务初始化（套餐定价、首批租户开通）
         │  └ 硬件对接（小票打印机现场调试）
         │
T-3 天   ┼─ 全流程验收（无障碍真机测试、支付小额闭环、通知闭环）
         │
T-0      ┴─ 正式上线
```

各阶段责任与并行关系：

| 阶段 | 主责 | 并行的事 | 串行依赖 |
|---|---|---|---|
| 1 | 商务 + 后端 | 与第 2/3 阶段并行 | 无（最先启动） |
| 2 | 运维 | 与第 1 阶段并行 | 无 |
| 3 | 后端 + 运维 | — | 依赖阶段 2 |
| 4 | PM + 店长 | 与第 5 阶段并行 | 依赖阶段 2、3 |
| 5 | 硬件 + 运维 | 与第 4 阶段并行 | 依赖阶段 2（CS 端能跑） |
| 6 | QA + 后端 | — | 依赖阶段 1-5 全部完成 |
| 7 | PM + 运维 | — | 依赖阶段 6 通过 |

## 2. 第一阶段：第三方账号申请（先启动，有审核周期）

### 2.1 微信小程序

**责任**：商务申请 + 后端配置 + 小程序开发
**耗时**：商务 0.5d；后端 0.5h；微信审核 1-3 工作日

#### 步骤

1. **商务**：登录 https://mp.weixin.qq.com 注册企业小程序账号，需准备：
   - 营业执照
   - 法人身份证
   - 对公账户（用于打款验证）
   - 小程序名称（建议：「XX 盲人按摩」或店铺品牌名）
2. **商务**：注册完成后，在「开发管理 → 开发设置」中获取 **AppId** 和 **AppSecret**，妥善保存（AppSecret 仅展示一次，建议拍照存到密码管理器）。
3. **后端**：把凭证写入生产环境配置（**不要提交到 git**）。
   修改 `backend/src/MassageSaas.Api/appsettings.Production.json`（或通过环境变量）：

   ```jsonc
   {
     "Notifications": {
       "WeChat": {
         "Enabled": true,
         "AppId": "wx1234567890abcdef",       // ← 替换
         "AppSecret": "your-app-secret-here",  // ← 替换
         "Page": "pages/index/index",
         "MiniProgramState": "formal"          // 正式版用 formal，体验版 trial
       }
     }
   }
   ```

   或使用环境变量（推荐生产环境）：
   ```
   Notifications__WeChat__AppId=wx1234567890abcdef
   Notifications__WeChat__AppSecret=your-app-secret-here
   Notifications__WeChat__Enabled=true
   ```
4. **小程序开发**：修改 `frontend/apps/miniprogram/project.config.json` 的 `appid` 字段为正式 AppId。
5. **运维**：在微信公众平台「开发 → 开发管理 → 开发设置 → 服务器域名」配置：
   - **request 合法域名**：填入后端 API 域名（必须 HTTPS，例如 `https://api.your-domain.com`）
   - 注意：每月只能修改 5 次，确认好再提交。

#### 验收
- 小程序开发者工具点击「编译」，无「未配置 AppId」报错。
- 真机调试模式下能完成 `wx.login()`。

### 2.2 微信订阅消息模板（4 个）

**责任**：后端配置
**耗时**：2h（不含每个模板的 1-7 天审核）

#### 步骤

1. 微信公众平台「订阅消息 → 我的模板 → 从公共模板库选用」，依次申请以下 4 个模板。模板字段需与代码字段名一致：

   | 用途 | Key | 必须字段（按代码生成的内容） |
   |---|---|---|
   | 会员生日提醒 | `MemberBirthday` | `name1` 姓名、`time2` 生日、`thing3` 祝福语 |
   | 会员卡到期提醒 | `MemberPackageExpiring` | `name1` 姓名、`thing2` 套餐名、`time3` 到期日 |
   | 预约前 30 分钟提醒 | `AppointmentReminder` | `name1` 姓名、`time2` 时间、`thing3` 门店、`thing4` 服务 |
   | 充值到账提醒 | `RechargeArrived` | `name1` 姓名、`amount2` 金额、`amount3` 余额 |

   > **注意**：实际字段名以微信公共模板库为准，订阅成功后获得「模板 ID」（形如 `xxxxxxxx_yyyyyyyy`）。如果公共模板库的字段与代码不匹配，需要在 `MassageSaas.Infrastructure/Notifications/` 调整字段映射。

2. **后端**：把 4 个模板 ID 填入配置：
   ```jsonc
   {
     "Notifications": {
       "WeChat": {
         "Templates": {
           "MemberBirthday":         "tpl_id_birthday_xxx",
           "MemberPackageExpiring":  "tpl_id_expiring_yyy",
           "AppointmentReminder":    "tpl_id_appoint_zzz",
           "RechargeArrived":        "tpl_id_recharge_www"
         }
       }
     }
   }
   ```

3. 重启 API。`GET /api/wechat/subscribe-templates` 应返回这 4 个 key。

#### 验收
- 小程序打开会员页 → 弹出「订阅消息授权」弹窗 → 用户同意后，下次充值能收到模板消息。
- 检查后端日志，无 `43101` 错误（43101 = 未授权）。

### 2.3 微信支付 V3

**责任**：商务申请 + 后端配置
**耗时**：商务 1d；后端 1h；商户号审核数个工作日

#### 步骤

1. **商务**：登录 https://pay.weixin.qq.com 申请「企业商户号」，需要：
   - 营业执照、对公账户、法人身份证（与小程序主体一致）
   - 申请「JSAPI 支付」+「APP 支付」（小程序内调起支付走 JSAPI）
   - 与小程序 AppId 绑定（在商户号后台「产品中心 → AppID 账号管理」中添加）
2. **商务**：审核通过后，在商户后台获取：
   - **商户号**（MchId，10 位数字）
   - **API v3 密钥**（在「账户中心 → API 安全 → 设置 APIv3 密钥」，**32 位字符串，自行生成并妥善保存**）
   - **商户 API 证书**（下载 `apiclient_cert.pem`、`apiclient_key.pem`，妥善保存）
   - **平台证书**（用于验签微信回调，通过 API 获取，证书会定期轮换）
3. **后端**：把凭证放到生产服务器**安全路径**（如 `/etc/massage-saas/wechatpay/`），权限 `600`，**严禁入库**。配置项（待 P2 真实支付集成补全；当前 `CallbackController` 已留位）：
   ```jsonc
   {
     "Payments": {
       "Wechat": {
         "MchId": "1234567890",
         "AppId": "wx1234567890abcdef",
         "ApiV3Key": "your-32-char-apiv3-key-xxxxxxxx",
         "MerchantCertPath": "/etc/massage-saas/wechatpay/apiclient_cert.pem",
         "MerchantKeyPath":  "/etc/massage-saas/wechatpay/apiclient_key.pem",
         "NotifyUrl": "https://api.your-domain.com/api/callback/wechat"
       }
     }
   }
   ```
4. **后端**：在微信商户后台「产品中心 → 开发配置」中，把支付结果回调 URL 设为 `https://api.your-domain.com/api/callback/wechat`（必须 HTTPS，公网可达）。
5. **后端**：确认 `/api/callback/wechat` 在 `TenantStatusMiddleware` 白名单内（已默认包含 `/api/callback/*`，无需改动）。

#### 验收
- 用真实手机扫码，0.01 元小额测试，订单状态变 `Paid`，且回调日志显示 `Tenant {id} renewed via Wechat`。
- 故意重复发送同一回调，第二次返回成功但不重复续期（幂等校验）。

### 2.4 支付宝

**责任**：商务申请 + 后端配置
**耗时**：商务 0.5d；后端 1h

#### 步骤

1. **商务**：登录 https://open.alipay.com 创建「网页&移动应用」，类型选「自研」，绑定营业执照。
2. **商务**：审核通过后，在「应用详情 → 开发设置」中：
   - 复制 **AppId**
   - 生成 **应用密钥**（推荐用「支付宝开放平台开发助手」一键生成 RSA2 密钥对）
     - 上传**应用公钥**到平台
     - 平台返回**支付宝公钥**
   - 添加产品：「手机网站支付」或「APP 支付」
3. **后端**：配置：
   ```jsonc
   {
     "Payments": {
       "Alipay": {
         "AppId": "2021000000000000",
         "PrivateKeyPath": "/etc/massage-saas/alipay/app_private_key.pem",
         "AlipayPublicKeyPath": "/etc/massage-saas/alipay/alipay_public_key.pem",
         "SignType": "RSA2",
         "Gateway": "https://openapi.alipay.com/gateway.do",
         "NotifyUrl": "https://api.your-domain.com/api/callback/alipay"
       }
     }
   }
   ```
4. **商务**：在支付宝开放平台「应用详情 → 开发设置 → 接口加签方式」配置异步通知地址（同上）。

#### 验收
- 0.01 元小额测试，回调返回 `success`，订单变 `Paid`。

## 3. 第二阶段：基础设施与部署

### 3.1 服务器与网络

**责任**：运维
**耗时**：1d

#### 推荐配置

| 环境 | CPU | 内存 | 磁盘 | 备注 |
|---|---|---|---|---|
| 生产-小型（10 店内） | 2 核 | 4 GB | 100 GB SSD | 单机 |
| 生产-中型（10-50 店） | 4 核 | 8 GB | 200 GB SSD | API + DB 拆开 |
| 生产-大型（50 店+） | 8 核 | 16 GB | 500 GB SSD | API 双节点 + 独立 DB |

#### 操作

1. 购买服务器（阿里云 ECS / 腾讯云 CVM / 自建），操作系统建议 Ubuntu 22.04 LTS 或 Windows Server 2022（CS 端打包不影响服务器选型）。
2. 安全组开放：80、443、22（限制源 IP），**3306/6379 不要对公网开放**。
3. 申请域名（如 `api.your-domain.com` 用于 API、`admin.your-domain.com` 用于平台端、`shop.your-domain.com` 用于按摩店 BS 端）。
4. 配置 DNS A 记录指向服务器 IP。
5. 申请 SSL 证书（推荐 Let's Encrypt 免费证书 + certbot 自动续期；或购买商业证书）。
6. 部署反向代理（Nginx 或 Caddy），HTTPS 终结在反代，转发到 API 的 HTTP 端口。

> **强制要求**：微信小程序、微信支付回调、支付宝回调**必须 HTTPS**。HTTP 不通过。

### 3.2 MySQL 8.0

**责任**：运维
**耗时**：2h

```bash
# Ubuntu 安装示例
sudo apt update
sudo apt install mysql-server-8.0
sudo mysql_secure_installation

# 登录 root 创建库与账号
sudo mysql -uroot
CREATE DATABASE massage_saas CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'massage'@'%' IDENTIFIED BY '强密码请生成32位以上';
GRANT ALL PRIVILEGES ON massage_saas.* TO 'massage'@'%';
FLUSH PRIVILEGES;
```

#### 关键配置

修改 `/etc/mysql/mysql.conf.d/mysqld.cnf`：

```ini
[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
max_connections = 200
innodb_buffer_pool_size = 2G    # 内存的 50%-70%
slow_query_log = 1
long_query_time = 1
```

#### 连接串配置

不要修改 `appsettings.json`（仓库内默认值给本地开发用）。生产环境用环境变量或 `appsettings.Production.json`：

```
ConnectionStrings__MySql=Server=10.0.0.5;Port=3306;Database=massage_saas;User=massage;Password=xxx;CharSet=utf8mb4;
```

#### 数据库迁移

首次启动 API，会自动执行迁移（`Database:RunMigrationsOnStartup: true`），自动 seed：
- 3 个套餐：基础版 ¥1980/年、标准版 ¥4980/年、旗舰版 ¥12800/年
- 1 个平台管理员账号：`admin / admin@123`（**首次登录立即改密**，见第 4 节）

> **手动迁移**（如需关闭自动迁移）：
> ```bash
> cd backend
> dotnet ef database update --project src/MassageSaas.Infrastructure --startup-project src/MassageSaas.Api
> ```

### 3.4 后端 API 部署

**责任**：运维
**耗时**：2h

#### 构建发布包

在开发机或 CI：
```bash
cd backend
dotnet publish src/MassageSaas.Api -c Release -o ./publish --self-contained false
```

把 `./publish` 目录传到服务器 `/opt/massage-saas/api/`。

#### 配置 systemd（Linux）

`/etc/systemd/system/massage-api.service`：
```ini
[Unit]
Description=Massage SaaS API
After=network.target mysql.service redis-server.service

[Service]
WorkingDirectory=/opt/massage-saas/api
ExecStart=/usr/bin/dotnet /opt/massage-saas/api/MassageSaas.Api.dll
Restart=always
RestartSec=10
User=www-data
Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=ASPNETCORE_URLS=http://127.0.0.1:5000
EnvironmentFile=/etc/massage-saas/env

[Install]
WantedBy=multi-user.target
```

`/etc/massage-saas/env`（敏感配置，权限 600）：
```
ConnectionStrings__MySql=Server=...;
Jwt__SecretKey=...
Notifications__WeChat__AppId=...
Notifications__WeChat__AppSecret=...
```

启动：
```bash
sudo systemctl daemon-reload
sudo systemctl enable massage-api
sudo systemctl start massage-api
sudo systemctl status massage-api
curl http://127.0.0.1:5000/health    # 应返回 200 + JSON
```

#### Nginx 反代配置

```nginx
server {
    listen 443 ssl http2;
    server_name api.your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/api.your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.your-domain.com/privkey.pem;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name api.your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 4. 第三阶段：安全加固

> ⚠️ **务必在上线前完成。**

### 4.1 替换 JWT 签名密钥

**责任**：后端
**耗时**：0.5h

仓库里 `appsettings.json` 的 `Jwt.SecretKey` 是开发用占位（明文 `dev-secret-key-change-me-...`），**生产必须替换**。

```bash
# 生成 64 字节强随机
openssl rand -base64 64
```

写入生产环境变量：
```
Jwt__SecretKey=生成出来的64字节base64串
Jwt__Issuer=MassageSaas
Jwt__Audience=MassageSaas.Client
Jwt__AccessTokenMinutes=120
Jwt__RefreshTokenDays=30
```

**密钥泄露 = 任何人可伪造任意用户登录**。

---

### 4.2 修改默认管理员密码

**责任**：PM 或运维
**耗时**：0.2h

首次启动 API 后，DbInitializer 会创建：

### 6.4 来电显示（已就绪，按需开启）

**责任**：硬件 + 运维
**耗时**：0.5h / 店

代码实现：`Services/Devices/Serial/SerialCallerIdMonitor.cs`，识别主流来电盒 ASCII 协议（USR / 联通来电小秘 / 56k modem，振铃后串口输出 `NMBR=13800138000\r\n`）。

#### 步骤
1. **硬件**：现场接来电盒到电话线 + RS232，记下串口号（如 `COM3`）和波特率（来电盒多数 `1200`）。
2. **运维**：编辑 `callerId` 节：
   ```jsonc
   "callerId": {
     "enabled": true,
     "serialPort": "COM3",
     "baudRate": 1200
   }
   ```
3. **运维**：重启 CS 端。CS 端启动后后台线程一直监听串口；来电时自动语音播报「来电，号码 138...」。
4. 故障：串口异常会指数退避自动重连（2s → 30s 上限），日志关键字 `来电串口异常`。

### 6.5 磁条读卡器（已就绪，按需开启）

**责任**：硬件 + 运维
**耗时**：0.5h / 店

代码实现：`Services/Devices/Serial/SerialCardReader.cs`，识别 ISO Track2 标准（`;CARD=DISC?LRC\r\n`，剥掉哨兵后取卡号）和简化数据流（纯数字 + 回车）。

#### 步骤
1. **硬件**：现场接磁条读卡器到 RS232，记下串口号（如 `COM4`）。
2. **运维**：编辑 `cardReader` 节：
   ```jsonc
   "cardReader": {
     "enabled": true,
     "serialPort": "COM4",
     "baudRate": 9600
   }
   ```
3. **运维**：重启 CS 端。在收银台界面刷卡：自动调出会员（卡号会写入「会员搜索」框并触发查询）；不在收银台时语音提示「请先切换到收银台再刷卡」。
4. 故障：日志关键字 `磁条读卡器串口异常`。

> **关于 USB HID 键盘式读卡器**：那种刷卡时直接「打字」到当前焦点，相当于人手敲数字 + 回车，CS 端的「会员搜索」框默认聚焦时就能直接吃进去——**不需要配置任何东西**，也不需要把 `cardReader.enabled` 改成 true。但这种模式刷卡数据会被任何聚焦控件吃掉，盲人收银员用读屏切换焦点时容易冲突；推荐用 RS232 串口模式。

## 附录 B：常见故障排查

| 现象 | 可能原因 | 排查 |
|---|---|---|
| API 启动失败 | 数据库连不上 | `curl http://127.0.0.1:5000/health`，看 `database` 节状态；检查连接串和 MySQL 权限 |
| 登录返回 401 | 密码错 / 账号被禁 | 查 `users` 表 `is_active`；用 BCrypt 验密 |
| 平台账号能登录但无菜单 | role claim 缺失 | 解 JWT 检查 `role`；DB 中 `users.role = 0` 是 PlatformAdmin |
| 写请求返回 403 + `TenantExpired` | 租户到期被软锁 | 平台端查租户 `expire_at`，做续费或线下激活 |
| 微信支付回调收不到 | 回调 URL 不可达 / HTTPS 证书问题 | `curl -X POST https://api.../api/callback/wechat`；查 Nginx 日志 |
| 微信支付回调 200 但订单未变 | 金额校验失败（前后端单位不一致） | 看 API 日志找 `AmountMismatch`；微信用「分」，本系统用「元」 |
| 订阅消息 43101 | 用户未授权 | 引导用户在小程序里重新点击触发 `wx.requestSubscribeMessage` |
| 订阅消息 40001 | AppSecret 错或缓存的 access_token 过期 | 检查 AppSecret；access_token 由代码自动刷新 |
| CS 端打印「测试打印小票」失败 | 端口被占用 / 串口号错 / 网络不通 | 看弹出的错误原因；用串口调试助手测端口 |
| 客显不亮 / 不刷新 | 串口号错 / 波特率错 / 编码不支持 | CS 端日志查 `客显写入失败`；用串口调试助手发 `ESC @` 测试 |
| 来电不播报 | 串口号错 / 来电盒不输出 NMBR= 格式 | 日志查 `来电串口异常`；用串口调试助手监听 COM 看原始数据 |
| 刷卡不调出会员 | 不在收银台界面 / 串口号错 / 用了 HID 模式但启用了 cardReader.enabled=true | 切回收银台再刷；检查 `磁条读卡器串口异常` 日志 |
| CS 端启动报错「无法连接 API」 | apiBaseUrl 配错 / HTTPS 证书在 Windows 不信任 | `curl` 测 API；HTTPS 证书要让 Windows 信任根 CA |
| 小程序登录失败 | AppId 不匹配 / 域名未加白名单 | 检查 `project.config.json` 和微信后台 request 合法域名 |
| 健康检查 `database` Unhealthy | DB 连不上 / 连接池打满 | 看 MySQL 状态；调大 `max_connections` 或 EF Core 连接池 |
| 报表数据不全 | 当前用户 `store_id` 限制了视图 | 店主/店长无 store_id 看全店；技师/收银员仅看本店 |
