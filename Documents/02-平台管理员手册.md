# 平台管理员手册

> 适用对象：SaaS 平台方运营人员（PlatformAdmin 角色）。
> 系统入口：平台端 Web —— `https://admin.your-domain.com`（生产）/ `http://localhost:5173`（开发）。
> 默认账号：`admin / admin@123`（**首次登录立即改密**）。
>
> 配套文档：《01-上线实施流程手册.md》《07-常见问题与故障处理.md》。

## 1. 你的职责

平台管理员负责"对外出租这套 SaaS 给按摩店"的全部生意侧动作：

- 新按摩店来谈生意 → 平台端开租户 → 收年费 → 续费/到期/欠费
- 出报表给老板看：哪家店赚钱、哪家店欠费、本月新签了几家
- 套餐价格调整、做活动、开发票

**你看不到任何一家门店的业务数据**（订单/会员/技师工资等）—— 那是按摩店自己运营的事。你只看见聚合数据。

## 2. 登录与改密

1. 浏览器打开 `https://admin.your-domain.com`。
2. 用户名 `admin`、密码 `admin@123`（默认）登录。
3. **立即**进入「个人设置 → 修改密码」改成强密码（≥12 位 + 大小写 + 数字）。
4. 上线前必做一次：在 DB 新建至少一个以负责人姓名命名的 PlatformAdmin 账号；再用 `admin` 登录禁用自己。详见《01-上线实施流程手册.md》第 4.2 节。

## 3. 主菜单

侧栏 4 项：

| 菜单 | 路径 | 用途 |
|---|---|---|
| 运营大盘 | `/` | 平台总览：租户总数、活跃数、本月新签、本月收入 |
| 营收报表 | `/revenue` | 按月/年统计平台收款、按套餐分布 |
| 按摩店租户 | `/tenants` | 新开/查看/续费/激活/暂停每家按摩店 |
| 套餐管理 | `/plans` | 三档套餐（基础/标准/旗舰）的价格、配额 |

## 4. 新开一家按摩店租户

> 一般你已经线下谈完合同了，店主把钱打过来，下面是开通流程。

**侧栏 → 按摩店租户 → 「新建租户」**：

1. **租户名**：填店铺正式名称（如「明心盲人按摩 · 朝阳店」）。会在租户自己的系统里到处显示。
2. **套餐**：从下拉里选基础/标准/旗舰。决定门店数与员工数上限。后期可改套餐（会重新计算配额）。
3. **店主账号**：
   - 用户名：建议 `店名拼音首字母`（如 `mxmr-admin`）
   - 临时密码：生成强随机串，交付给店主后他自己改
4. **保存** → 系统创建租户记录 + 一个 ShopOwner 角色账号 + 一家默认门店壳子。
5. 把账号 / 临时密码 / 登录 URL（`https://shop.your-domain.com`）交给店主，告诉他第一件事是改密。

> 平台端**不替店主录任何业务数据**（门店、技师、服务项目、价格等）—— 这些归店主自己做，参考《03-店主店长手册.md》第 3 章「初次开店：业务初始化清单」。

## 5. 续费与到期管理

每家租户的 `expire_at`（到期时间）决定写权限：

- **未到期**：正常运营
- **到期**（`status = Expired`）：仍能登录、能查任何数据（读），但**所有写请求被中间件 `TenantStatusMiddleware` 拦截返回 403**。例外白名单：登录 / 续费 / 回调 / 健康检查
- **暂停**（`status = Suspended`）：人工锁定，平台决定（如违约处理）。表现同到期

### 5.1 走在线支付续费（推荐）

店主在自己 BS 端的「订阅状态」里点续费 → 微信/支付宝扫码 → 回调到账后自动续期。**全程不用你介入**。

### 5.2 走线下汇款激活

适用：大额转账 / 客户不愿走在线支付 / 微信支付商户号未就绪。

**侧栏 → 按摩店租户 → 找到那一行 → 「线下激活」**：

1. 选续费时长（1 年 / 2 年 / 3 年）
2. 填备注（汇款单号、汇款日期、经办人）
3. 确认 → 后台调 `POST /api/subscriptions/activate-offline` → 该租户 `expire_at = max(now, expire_at) + 时长`

> **重要**：线下激活前**必须**确认款已到账。系统不会校验汇款真伪，激活操作不可撤销。

### 5.3 续费提醒看哪

「按摩店租户」列表的「到期日」列：

- 距到期 ≤ 30 天：黄色高亮
- 距到期 ≤ 7 天：红色高亮
- 已到期：灰色 + "已停用写功能"标签

可以筛选下「快到期」批量电话催费。

## 6. 营收报表

**侧栏 → 营收报表**：

- 上方时间筛选：本月 / 上月 / 今年 / 自选区间
- 总收款 = 该期间内 `payment_orders.status = Paid` 的金额合计（含微信/支付宝/线下）
- 按套餐分布饼图：知道哪一档卖得多，是不是该调价
- 续费 vs 新签柱状图：续费率低说明产品有问题，新签下滑说明销售环节有问题
- 详单可导出 CSV（用于交财务）

> 后端口径：`GET /api/dashboard/platform/revenue`，统计来源是 `payment_orders` 表，**所有回调幂等**（同一个微信 `out_trade_no` 重放只入账一次）。

## 7. 单租户运营概览

「按摩店租户」列表 → 任意一行右边 → 「运营概览」按钮：

弹出 `TenantOverviewDialog`，展示：

- **规模**：门店数 / 在职技师数 / 累计会员数
- **本月营业额**：所有门店求和（你能看到聚合数，但点不进去看具体订单）
- **技师营收 Top 10**：按本月提成排序

这个面板用于：

- 销售复盘：哪家店做得好可以请来当样板
- 续费议价：续费前看看人家做得怎么样
- 风险预警：营业额连续掉的店要及时联系

## 8. 套餐管理

**侧栏 → 套餐管理**：

DbInitializer 默认 seed 三档：

| code | 名称 | 门店数上限 | 员工数上限 | 年费 |
|---|---|---|---|---|
| basic | 基础版 | 1 | 10 | ¥1980 |
| standard | 标准版 | 3 | 30 | ¥4980 |
| enterprise | 旗舰版 | 10 | 200 | ¥12800 |

**编辑价格**：直接在列表里改 → 保存。新签的租户按新价收，已有租户**保持原价直到下一次续费**。

**新增套餐**：右上「新增」→ 设 code / 名称 / 配额 / 年费 → 保存。code 一旦使用过不可改。

**禁用套餐**：勾选 `disabled` → 下次新签时不再出现在选项里。已有租户不受影响。

> 套餐变动**不改代码、不改数据库结构**。
> 实际配额校验逻辑：店主新开门店或新增员工时，系统读 `tenant.plan_id` → 比对 `plan.maxStores / plan.maxStaff` → 超限拒绝。

## 9. 支付通道运维

平台只在「上线时」配置一次第三方凭证。日常运维你不碰，但出问题要会查：

### 9.1 配置项

在生产服务器 `/etc/massage-saas/env`（systemd EnvironmentFile）：

```
Payments__Wechat__MchId=...
Payments__Wechat__ApiV3Key=...
Payments__Alipay__AppId=...
Payments__Alipay__PrivateKeyPath=...
```

凭证申请流程见《01-上线实施流程手册.md》第 2.3 / 2.4 节。

### 9.2 回调失败排查

店主反馈"我付了钱怎么没续期"，按这个顺序查：

1. 「按摩店租户」找到该租户 → 「查看订单」→ 找到对应的 `payment_orders` 行：
   - 状态 `Pending` → 钱根本没到（用户中途取消 / 网络问题）
   - 状态 `Paid` 但 `expire_at` 没动 → 回调到了但续期逻辑错（找开发）
2. 看 API 日志（`logs/`）找 `out_trade_no=xxx` 的处理记录：
   - 没有这条日志 → 微信侧没回调（确认回调 URL 在公网可达 + HTTPS 正常）
   - 有 `AmountMismatch` → 金额单位错（微信用分、本系统用元）
   - 有 `SignatureFailed` → APIv3 密钥或证书错
3. 详细排查见《07-常见问题与故障处理.md》「支付」章节。

### 9.3 重放回调

如果确认钱到了但订单状态没变，可以从微信商户后台手动重发一次回调。**永远不要直接改库**——回调逻辑保证幂等，重放是安全的；改库会留下数据漏洞。

## 10. 数据安全与合规

- 平台管理员**没有**任何按摩店内部业务表的访问 UI（订单/会员/技师/工资）。底层数据库虽然你能看，但**不要从 DB 直接查门店数据出报告或发给第三方**——违反合同。
- 续费/激活/暂停操作都有审计日志（`audit_logs` 表）：操作人、时间、目标租户、变更前后。任何争议查这里。
- 不要把 PlatformAdmin 账号交给销售/客服。客服级别的查询账号需要单独设计（当前未实现）。

## 11. 系统状态自检

- **健康检查**：`https://api.your-domain.com/health` → 应返回 `{"status":"Healthy"}`。监控系统应每 5 分钟探测一次（《01-上线实施流程手册.md》第 9.1 节）。
- **后端日志**：`/opt/massage-saas/api/logs/`（生产服务器）。出问题第一时间看这里。
- **数据库备份**：每天 0:00 自动备份到 `/backup/`，保留 30 天 + 同步到对象存储。每月最少演练一次恢复。

## 12. 用户管理职责边界

| 操作 | 谁来做 |
|---|---|
| 新增 PlatformAdmin 账号 | 平台管理员（在 DB 直接 INSERT 或开发提工单） |
| 新增门店店主账号 | 平台管理员（开租户时一起建） |
| 门店内部账号（店长/收银员/技师） | 店主自己在 BS 端「员工管理」建 |
| 重置店主密码 | 平台管理员（在「按摩店租户」详情页 → 「重置店主密码」），生成新临时密码交付 |
| 重置门店内部账号密码 | 店主自己 |

## 13. 常见操作速查

| 任务 | 在哪点 |
|---|---|
| 新开按摩店租户 | 按摩店租户 → 新建租户 |
| 续费/激活到期租户 | 按摩店租户 → 选行 → 线下激活 |

| 查本月平台收入 | 营收报表 → 选「本月」 |

| 自己改密码 | 右上头像 → 个人设置 → 修改密码 |
| 退出登录 | 右上头像 → 退出 |

## 14. 出问题找谁

- **登录不上 / 系统报错 / 数据异常** → 找运维 + 看《07-常见问题与故障处理.md》
- **支付通道故障** → 找后端开发 + 微信/支付宝商户后台对单
- **店主投诉功能不好用** → 收集详细复现步骤 + 截图 + 时间，转给产品经理；技术 Bug 转开发
- **新增需求** → 提需求池，不要承诺工期
